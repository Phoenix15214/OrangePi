# 串口发送内容

| 序号 |      内容       |                          备注                          |
| :--: | :-------------: | :----------------------------------------------------: |
|  1   |  循迹偏离角度   |          为了避免发送负数，每个数据均加了180           |
|  2   | 循迹偏离中心x值 | 循迹线在中心时为0， 为了避免发送负数，每个数据加了1000 |

|  4   |    路口y坐标    |       路口处的y坐标，是路口在摄像头中的绝对y坐标       |
|  5   |   数字识别ID1   |   没有识别到目标时为0，识别到时显示具体识别到的数字    |
|  6   |   数字识别ID2   |                          同上                          |
|  7   |   数字识别ID3   |                          同上                          |
|  8   |   数字识别ID4   |                          同上                          |
|  9   |    终点x坐标    |       终点处的x坐标，是终点在摄像头中的绝对x坐标       |
|  10  |    终点y坐标    |       终点处的y坐标，是终点在摄像头中的绝对y坐标       |

发送数据总量为10，包头为0x14

## 其他注意事项

- 循迹时摄像头分辨率为320x240

- 因路口坐标是计算得出，坐标可能超出摄像头分辨率范围
- 终点通过识别终点处的多个黑色色块实现，得到的x、y坐标为黑色色块群整体的中心坐标，所以在接近终点导致终点无法识别时，y坐标值可能不会接近分辨率边界
- 数字识别的ID顺序为从左到右
